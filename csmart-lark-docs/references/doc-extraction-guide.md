# 需求文档提取指南

本文档说明如何从 PRD/需求文档中提取测试所需的业务规则。

## 提取目标

从需求文档中提取以下业务规则：

### 1. 触发词 (triggers)

**搜索关键词**：触发词、关键词、trigger、指令、命令

**提取规则**：
- 识别用户发送消息时使用的关键词
- 包括主触发词和别名
- 注意中英文变体

**示例**：
```json
{
  "triggers": ["安全相", "巡火", "巡夜"]
}
```

### 2. 点位字典 (points)

**搜索关键词**：点位、位置、point、location、区域、zone

**提取规则**：
- 识别所有有效的点位编号
- 按区域分组（如有）
- 注意特殊点位（如 AM、CUR、BM1 等非数字点位）

**示例**：
```json
{
  "zones": {
    "BLK A": ["A16", "A15", "A14", "...", "A1", "AM", "AG", "A0"],
    "BLK B": ["B18", "B17", "B16", "...", "B1", "BM", "BG", "B0"],
    "BLK C": ["CUR", "CMR", "C18", "...", "C1", "CG", "C0"],
    "地牢": ["BM1", "BM2"],
    "外围": ["E"]
  }
}
```

### 3. 调度规则 (schedule)

**搜索关键词**：定时、自动、schedule、cron、每小时、每天

**提取规则**：
- 识别定时任务的 cron 表达式
- 记录调度时间和频率
- 注意时区设置

**示例**：
```json
{
  "schedule": "cron 0 18-23,0 * * *",
  "description": "18:00 至 00:00 每小时自动总结"
}
```

### 4. 业务日规则 (business_day)

**搜索关键词**：业务日、跨日、零点、重置、business day、reset

**提取规则**：
- 识别业务日的开始时间
- 记录跨日重置规则
- 注意时区偏移

**示例**：
```json
{
  "business_day_start": 6,
  "description": "业务日从 06:00 开始，零点后重置状态"
}
```

### 5. 输入格式 (input_format)

**搜索关键词**：输入、格式、format、连接符、分隔符

**提取规则**：
- 识别用户输入的格式规范
- 记录支持的连接符和分隔符
- 注意特殊字符处理

**示例**：
```json
{
  "separators": ["-", "~", "～", "—", "——"],
  "format": "触发词 + 空格 + 点位范围（如：安全相 A1-A10）",
  "examples": ["安全相 A1", "巡火 A1-A10", "安全相 B1~B5"]
}
```

### 6. 输出格式 (output_format)

**搜索关键词**：输出、回复、response、消息、总结

**提取规则**：
- 识别系统的回复格式
- 记录总结消息的结构
- 注意特殊符号（如 ✅、❌）

**示例**：
```json
{
  "summary_format": "按区域显示已巡（✅）和未巡（❌）点位",
  "symbols": {
    "completed": "✅",
    "pending": "❌"
  }
}
```

## 提取流程

### 1. 读取文档

使用 lark-mcp 读取文档内容：

```bash
curl -X POST http://localhost:3456/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": {
      "name": "docx_v1_document_rawContent",
      "arguments": {
        "path": {
          "document_id": "<DOC_TOKEN>"
        },
        "useUAT": true
      }
    }
  }'
```

### 2. 搜索关键词

在文档内容中搜索上述关键词，定位相关章节。

### 3. 提取规则

从相关章节中提取具体的业务规则。

### 4. 验证规则

- 检查提取的规则是否完整
- 验证规则之间是否冲突
- 确认规则的格式正确

## 常见文档结构

### 巡火类文档

通常包含：
- 触发词列表
- 点位字典（按区域分组）
- 连接符和分隔符
- 调度规则（cron 表达式）
- 总结消息格式

### E-permit 类文档

通常包含：
- 许可类型（纸本/E-permit）
- 申请流程
- 审批规则
- 状态流转
- 通知规则

### 纸本许可类文档

通常包含：
- 许可类型
- 申请材料
- 审批流程
- 有效期规则
- 提醒规则

## 输出格式

将提取的规则整理为 JSON 对象：

```json
{
  "triggers": ["安全相", "巡火"],
  "zones": {
    "BLK A": ["A16", "A15", "...", "A0"],
    "BLK B": ["B18", "B17", "...", "B0"]
  },
  "schedule": "cron 0 18-23,0 * * *",
  "business_day_start": 6,
  "separators": ["-", "~", "～", "—", "——"],
  "summary_format": "按区域显示已巡和未巡点位"
}
```

## 注意事项

1. **权限问题**：如果 PRD 无权限，使用原始需求副本
2. **文档版本**：注意文档的最后修改时间，确保使用最新版本
3. **规则冲突**：如果文档中存在冲突的规则，记录并提示用户确认
4. **缺失规则**：如果文档中缺少某些规则，使用默认值或提示用户补充
